The use cases
What we heard on the 24th
Use Case 01
Regulator Investment
Plan Validator
A utility client submits investment plans to an energy regulator. The regulator accepts or rejects. Today the client submits and waits — with no visibility into whether the draft will land until the decision arrives.
The value is in the prediction — knowing before submission whether the document will be accepted, and exactly what is reducing the probability.
Use Case 02
Solution Design
Document Reviewer
An enterprise with a defined architecture standard needs every solution design reviewed against it before sign-off. Today that review depends on the availability of senior architects and produces inconsistent outcomes.
The value is consistency at scale — every design held to the same standard, every gap traced to a specific rule, without the bottleneck of manual review.
The architecture — Use Case 1
Regulator Investment Plan Validator
Zone 1 — Input & Rule Store
Historical Propositions
Previous investment plans — PDF, Word, email attachments
Evaluation Records
Regulator responses — acceptance and rejection letters, feedback correspondence, annotated returns
Rule Derivation
AI layer — induces rule set from submission outcomes; identifies acceptance and rejection patterns across the corpus
Rule Store
Living, versioned vector corpus — reviewable and updatable by human reviewer; grows as new submissions are ingested
Zone 2 — Evaluation Engine
New Investment Proposition
Uploaded document — evaluated against the derived rule set
Compliance Matching Agent
How well does this document satisfy the derived acceptance criteria?
Document Consistency Agent
Quantitative discrepancies, allocation conflicts, temporal contradictions, definitional drift across sections
Risk Pattern Detection Agent
Does this document contain language or omissions associated with historical rejections?
Scoring Engine
Combines compliance match, consistency audit, and risk pattern signals into a single probability estimate
Six-Layer Guardrail System
Rule derivation — no rules induced without evidential basis
Consistency audit — no issue flagged without cross-section evidence
Output generation — no recommendations beyond the observation register
Zone 3 — Output
Acceptance Probability Score
Single confidence figure with band — estimated likelihood of regulatory acceptance
Observation Register
Per-item findings that reduce the score — each traced to source rule and historical evidence
Improvement Recommendations
Per observation — specific action with estimated score impact if applied
The architecture — Use Case 2
Solution Design Document Reviewer
Zone 1 — Input & Rule Store
Enterprise Architecture Rule Set
Canonical EA standards, architecture principles, and non-negotiables — uploaded and maintained
Golden Templates optional
Representative compliant solution design documents — calibrate the engine's interpretation of rules in context
Rule Store
Versioned vector corpus of EA standards and templates — reviewable and updatable as standards evolve
Zone 2 — Evaluation Engine
Solution Design Document
Single document uploaded for review against the EA rule set
Requirements Coverage Agent
For each rule — satisfied, partially satisfied, or absent? Produces a coverage map by domain
Document Consistency Agent
Quantitative discrepancies, allocation conflicts, temporal contradictions, definitional drift across sections
Quality & Template Alignment Agent
Deviations from golden templates, undeclared omissions, structural gaps
Scoring Engine
Combines coverage, consistency audit, and template alignment signals into a weighted compliance score by domain
Six-Layer Guardrail System
Rule ingestion — only uploaded EA standards enter the rule store
Consistency audit — no issue flagged without cross-section evidence
Output generation — no recommendations beyond what the rule set supports
Zone 3 — Output
Compliance Score
Headline figure with domain breakdown — e.g. 74% of EA requirements satisfied across security, integration, data, infrastructure
Observation Register
Per-requirement findings — rule referenced, document section cited, gap severity indicated
Improvement Recommendations
Per observation — what to add, change, or remove, with estimated score impact
