Full Process Table — from supplier feed to a DPP-ready, audited product record

The compliance-led variant. BuildMart's wedge is the “silent break”: the PIM completeness check only verifies a field is filled, not that the value is valid — so a non-compliant Euroclass / DoP value can clear the PIM and reach the live site. This engine validates the value, per channel, checks certificates live against public registries, chases the gaps, and assembles a mandatory DPP payload — construction is an ESPR early-mover category, which turns “nice to have” into a regulatory-deadline buy. System (AI) and human steps are kept strictly separate; the In plain English column explains every step to a layman.

Key terms

Silent break — a non-compliant supplier value clears the PIM’s “is the field filled?” check and reaches the live site before anyone catches it. The pain this engine removes.
DoP / CE — Declaration of Performance & CE marking. The mandatory conformity basis for a construction product; every performance claim must trace to it.
Euroclass — the EU fire-reaction classification (e.g. A1…F). A regulated value that must match the DoP, not be invented in copy.
R-value / λ — thermal resistance and thermal conductivity. The headline performance numbers for insulation; jurisdiction-sensitive wording.
REACH / SVHC — EU chemicals regime; Substances of Very High Concern statement. A required DPP/safety data point.
CPR — Construction Products Regulation (EU 305/2011) + harmonised standard. The core jurisdictional rule set; PL transposes locally, UK diverges (UKCA).
ACERMI — French insulation certification with a public registry and an expiry cycle — the ideal “supplier-declared ≠ valid, check the register” live-validation demo.
ETIM — the technical product-classification nomenclature. Today the values arrive in free-text, non-scopable fields; the engine maps them so they become checkable.
ESPR / DPP — Ecodesign for Sustainable Products Regulation & the Digital Product Passport. Construction is a confirmed early-mover — a real deadline, and the wedge.
ESL — in-store electronic shelf labels. One of several downstream channels (with e-com, print catalogues, in-store screens, marketplaces) the same value feeds.

The iteration loops

Where the process cycles back. Each loop is a real workflow object (P21), not an email thread — held in the system until resolved.
1 · Value / cert chaseC1 / C2 / C3 ↔ B3 / B4
An invalid value or an expired certificate blocks the item until a valid one arrives, then re-validates automatically. This is the loop that closes the silent break.
2 · Completeness chaseB5 → C1 → … → B5
Missing DPP / passport fields are chased until the record is complete — the passport schema drives the gap list, so nothing ships half-described.
3 · Compliance amend (governed)D1 → B3
A regulated-wording amend re-runs value validation with the new citation; stays inside the governed track.
4 · Portfolio rejectD2 → B1 / B2
A channel-expression reject sends the variant back to re-generation with the reviewer’s note carried in as a hard constraint.
The wedge: the PIM’s completeness check confirms a field is filled; this engine confirms the value is valid (B3) and the certificate is current (B4). That is the “silent break” closed — the non-compliant Euroclass / DoP value caught before it fans out to e-com, catalogue, shelf label and marketplace.
Two desks: Compliance (D1, signs DoP/CE) owns the regulated core wording; Portfolio (D2) owns the channel expression. Confidence scores (B6) decide which desk sees what; P15 Confirm / Override / Query sits on every AI artefact.
DPP is the wedge, not a wrapper: construction is a confirmed ESPR early-mover with a genuine deadline and open internal ownership (F3) — the “nice to have” becomes a regulatory-deadline buy. New patterns: P26 rule-bounded multi-variant generation (B1/B2/E1); P27 live registry validation (B4) — supplier-declared ≠ valid.
Commercial frame: PoC ≈ €45–60k; success = N SKUs enriched + validated across FR/UK/PL with a full audit trail in 6–8 weeks; EU data residency + IT-led security review. [YI B.10]
Design decision inventory — why the process looks this way
The decisions that shaped the process above, each with its rationale, build implication, and source. [YI] / [TOV] = prospect input · [PL] / [REG] = pattern library / regulation · [O] = our design.
# Design decision Rationale Implication for the build Source
D1 Validate the value, not just completenessThe wedge against the “silent break”. The PIM’s completeness check only verifies a field is filled; a non-compliant supplier value clears it and reaches the live site. B3 does per-channel value matching (Euroclass / R / λ vs DoP); the differentiator is value-level, not presence. [YI B.4]
D2 Certificates are validated live against public registriesSupplier-declared ≠ valid. ACERMI, FSC, Keymark etc. expire on cycles; a declared status is not proof of a current certificate. B4 calls the public registry / API, is expiry-aware, and flags expired or mismatched certs. New pattern P27. [YI A.7]
D3 DPP is the wedge, mandatory and registeredLead with the regulatory-deadline buy. Construction is a confirmed ESPR early-mover with a real deadline and open internal ownership — a clean opening. B5 completeness gate against the DPP schema; F3 assembles + registers the passport. Not optional. [YI A.4/B.7] · [REG]
D4 Two desks: Compliance (DoP/CE) then PortfolioCore wording vs channel expression. Legal/regulatory validates the core wording against the framework; marketing/digital owns the online expression. D1 compliance review (signs DoP/CE); D2 binding portfolio sign-off; confidence (B6) routes each. [YI B.1]
D5 ETIM mapping makes attributes scopableFree-text today, checkable tomorrow. Claims arrive in free-text, non-scopable fields; a value can’t be validated until it is mapped to a known attribute. A3 maps free-text → ETIM as part of extraction, before any rule runs. [YI A.8]
D6 Sit beside the PIM as the compliance layerNo upstream PLM here. An Akeneo-class PIM is in place but only checks completeness; there is no PLM where claims originate. E2 push-back to PIM only; the engine adds the compliance layer the PIM lacks. [YI A.8]
D7 Model UK as an export / marketplace edge caseSharpens the online-vs-print split. This retailer’s UK footprint is export / marketplace only; UKCA diverges from CE post-Brexit. B2 renders UK as the edge variant, making “online allows more flexibility” visible against verbatim print. [YI framing]
